Diagrama de Flujo: Proceso de Gestión y Automatización INE

Integración de Webhooks, extracción OCR (Nano Banana), validación segura y scrapping automatizado.

Roles / Capas
Step 1: Captura, OCR e Ingesta
Step 2: Lógica y Orquestación
Step 3: Ejecución y Scraping
Step 4: Documentos y Notificación
Originador / Usuario
Solicitud de Consulta
Ingresa imagen de INE o datos mediante Webhook / Formulario.
Cierre de Proceso
Recibe la validación final (Vigente/Error) y archivo adjunto.
n8n (Orquestador)
Extracción OCR
Se envía imagen a Nano Banana para extracción de texto.
Limpieza Raw
Homologa las salidas del OCR (CIC, OCR, Clave Elector).
Detección de Modelo
Identifica modelo INE (A, B, C... H) a partir de los datos limpios.
Enrutamiento a API
Arma el JSON final y lo envía al sub-flujo / API en Python.
Motor de Correo
Detección de template (Exitoso/No Encontrado) y envío vía SMTP final.
API Python (INE)
Validación Token
Endpoint protegido verifica JWT mediante JWKS de Auth0.
Scraper Playwright
Cola segura (Lock) -> Resuelve 2Captcha -> Consulta Lista Nominal -> Retorna JSON.
API Utils (Mock)
Transformación
Endpoint `/pdf/image-to-pdf` toma la captura web (Base64) y retorna PDF estructurado.
Datos / Seguridad
Hashicorp Vault
Inyecta credenciales y tokens al entorno sin exponer variables de entorno.
Trazabilidad
Enmascaramiento de PII. Guarda petición (`ECIA_TIP_INE_REQUEST`) y respuesta.
